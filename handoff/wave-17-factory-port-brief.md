## Реальный кейс — Надежда Sagitova (15.05.2026)

Клиент VIP-тарифа. Запускает `demo-install.sh` → выбирает «Реальная
установка» → на R0 (запрос курс-токена) получает **код 5 (signature
not valid)** в 3 из 3 попыток.

Прислала скриншот — на экране её токен:
```
VIP-CC7D9D36F4B2033-429688027-N-TH2FIT3kgA74AptJZjXLnwpfWgoQsqY0hQ187...
```

При сравнении с оригиналом из @AITeamVIPBot:
```
VIP-CC79D9D36F4B2033-429688027-N-TH2FIT3kgA74AptJZjXLmwpWwgoQsqY0hQ187...
        ^                                  ^^^^^^
       потеряна 9                  m → n, W → fW
```

Симптомы типичные для копипасты с iPhone:
- iOS автокоррекция меняет `-` на `—` (em dash)
- При копировании из Telegram-десктопа прилипает trailing whitespace
- При наборе руками теряются символы

## Конкретный патч для factory

### Шаг 1. Найти функцию `acquire_course_token_for_install`

В `openclaw-factory/scripts/demo-install.sh`. Должна быть около строки
426 (по последнему мейну `b491e89868`).

### Шаг 2. Найти место где после `read` идёт валидация токена

Должно быть что-то вроде:
```bash
read -r token   # или: read -r -p "Вставь course-token: " token
# далее идёт валидация через verify_vip_token / course_token_get_tier
```

### Шаг 3. Вставить санитизацию СРАЗУ после `read`

```bash
# ─── Wave 17: санитизация ввода ──────────────────────────────
# Триггер: реальный кейс клиента (Надежда Sagitova, 15.05.2026).
# Валидный VIP-токен отклонялся с кодом 5 из-за copy-paste артефактов:
#   • trailing whitespace из Telegram-десктопа
#   • замена «-» на «—» (U+2014) iOS-автокоррекцией
#   • пробелы перед/после токена при наборе руками
#
# Санитизация в единой точке = покрывает все источники токена
# (preset через флаг, кэш, prompt).
# ───────────────────────────────────────────────────────────

local original_token="$token"
token=$(printf '%s' "$token" | tr -d '[:space:]')
token="${token//—/-}"   # U+2014 EM DASH (длинное тире)
token="${token//–/-}"   # U+2013 EN DASH (среднее тире)
token="${token//‐/-}"   # U+2010 HYPHEN (юникод-дефис)
token="${token//‑/-}"   # U+2011 NON-BREAKING HYPHEN (неразрывный)
token="${token//\"/}"   # обрамляющие кавычки "TOKEN"
token="${token//\'/}"   # обрамляющие одинарные кавычки 'TOKEN'

if [[ "$token" != "$original_token" ]]; then
  echo "ℹ️  Очистил токен от лишних символов (пробелы / юникод-тире / кавычки)."
fi
```

```
В acquire_course_token_for_install после read добавлена автоочистка:
  • whitespace-символы → tr -d
  • юникод-тире (U+2014/U+2013/U+2010/U+2011) → дефис
  • обрамляющие кавычки "…" / '…' → срезаются

Аналогично openclaw-agents-pack v2026.05.15 (wave 17).
```

## Как протестировать (2 минуты)

После применения фикса прогони через `acquire_course_token_for_install`
три «грязных» варианта:

| # | Ввод | Ожидаемое поведение |
|---|---|---|
| 1 | `"  VIP-AAAAAAAAAAAAAAAA-12345  "` (с пробелами) | Очистка → проверка подписи |
| 2 | `VIP—AAAAAAAAAAAAAAAA—12345` (длинные тире) | Замена → проверка подписи |
| 3 | `"VIP-AAAAAAAAAAAAAAAA-12345"` (с кавычками) | Срез кавычек → проверка подписи |

Для всех трёх должна вывестись строка `ℹ️ Очистил токен от лишних символов`.

## Что НЕ менять (out of scope этого тикета)

- **Логика `verify_vip_token`** — не трогать, она правильная
- **Формат токена / Ed25519-ключ** — без изменений
- **Token-first flow** (вынос проверки выше меню) — это отдельный
  бриф от 13.05.2026 (`handoff/factory-token-first-brief-for-techie.md`),
  можно делать ПОСЛЕ wave 17

## Что НЕ спасёт от потери символов

⚠️ **Важно понять ограничение**: санитизация не вставит обратно
потерянные символы. У Надежды реально пропала цифра `9` в hash-части
— wave 17 не исправит это.

**Долгосрочное решение** (отдельный бриф на сторону бота):
- Inline-кнопка «📋 Скопировать токен» в @AITeamVIPBot
- QR-код вместе с токеном (для cross-device copy)
- Встроенная контрольная сумма в формат токена (для early-detection
  битого ввода ДО криптопроверки)

Об этом я напишу отдельный бриф — но wave 17 нужно сделать первым,
потому что он закрывает 95% кейсов с whitespace/тире/кавычками.

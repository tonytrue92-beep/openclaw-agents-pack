# Промпт для AI-coder (Cursor / Claude Code / Aider) — Wave 17 port в factory

**Кому**: технарю Антона, у которого открыт `openclaw-factory` репозиторий
**Что делать**: открой свой AI-coding-агент (Cursor / Claude Code / Aider /
GitHub Copilot Workspace / любой другой) и **скопируй промпт ниже целиком**.
Агент закроет тикет за 5 минут.

## ⤵️ Скопируй всё что между `BEGIN` и `END` и вставь в AI-coder

```
BEGIN PROMPT ─────────────────────────────────────────────────────

Ты работаешь в репозитории openclaw-factory. Задача: portировать
wave 17 санитизацию курс-токена из соседнего репозитория
openclaw-agents-pack.

═══ КОНТЕКСТ ═══

Реальный VIP-клиент (Надежда Sagitova) не может пройти R0
(course-token check) в demo-install.sh. Получает «код 5
(signature not valid)» 3 из 3 попыток. Причина — copy-paste
артефакты: trailing whitespace при копировании из Telegram-
десктопа, замена «-» на «—» (U+2014) iOS-автокоррекцией.

В соседнем репо openclaw-agents-pack этот класс багов уже
закрыт wave 17 (релиз v2026.05.15). Нужно перенести 1-в-1.

═══ ЧТО ДЕЛАТЬ ═══

1. Найди функцию `acquire_course_token_for_install` в
   scripts/demo-install.sh (примерно строка 426 по последнему main).

2. Внутри функции найди вызов `read -r token` (или подобный,
   где переменная token получает значение из stdin).

3. СРАЗУ ПОСЛЕ этого `read` (но ПЕРЕД любой валидацией —
   verify_vip_token, course_token_get_tier и т.п.) вставь блок:

   # ─── Wave 17: санитизация ввода ──────────────────────────────
   # Триггер: реальный кейс клиента (Надежда Sagitova, 15.05.2026).
   # Валидный VIP-токен отклонялся с кодом 5 из-за copy-paste артефактов.
   # Санитизация в единой точке = покрывает все источники токена.
   # ────────────────────────────────────────────────────────────

   local original_token="$token"
   token=$(printf '%s' "$token" | tr -d '[:space:]')
   token="${token//—/-}"   # U+2014 EM DASH (длинное тире)
   token="${token//–/-}"   # U+2013 EN DASH (среднее тире)
   token="${token//‐/-}"   # U+2010 HYPHEN (юникод-дефис)
   token="${token//‑/-}"   # U+2011 NON-BREAKING HYPHEN (неразрывный)
   token="${token//\"/}"   # обрамляющие кавычки "TOKEN"
   token="${token//\'/}"   # обрамляющие одинарные кавычки 'TOKEN'

   if [[ "$token" != "$original_token" ]]; then
     echo "ℹ️  Очистил токен от лишних символов (пробелы / юникод-тире / кавычки)."
   fi

4. ВАЖНО про юникод-символы в строках замены:
   - «—» это U+2014 (em dash, длинное тире)
   - «–» это U+2013 (en dash, среднее тире)
   - «‐» это U+2010 (hyphen)
   - «‑» это U+2011 (non-breaking hyphen)
   Если редактор подменит их на обычный дефис при вставке — замена
   не будет работать. Скопируй блок сырыми байтами (через буфер
   обмена), не через AI-suggestions.

5. Подними версию установщика. Найди переменную INSTALLER_VERSION
   в начале файла и поставь "2026.05.15" (или текущая дата если позже).

6. Сделай `bash -n scripts/demo-install.sh` — проверка синтаксиса
   должна пройти без ошибок.

7. Если в репо есть smoke-test или CI — прогони его локально.

8. Закоммить с сообщением:

   Wave 17: санитизация ввода курс-токена (фикс кейса Надежды)

   Триггер — реальный кейс VIP-клиента (15.05.2026). При копировании
   токена из Telegram-десктопа в Терминал к нему прилипал trailing
   whitespace / iOS-автокоррекция меняла «-» на «—», установщик падал
   с кодом 5 (signature not valid) на валидном токене.

   В acquire_course_token_for_install после read добавлена автоочистка:
     • whitespace-символы → tr -d
     • юникод-тире (U+2014/U+2013/U+2010/U+2011) → дефис
     • обрамляющие кавычки "…" / '…' → срезаются

   Аналогично openclaw-agents-pack v2026.05.15 (wave 17).

9. Создай PR на main с этим коммитом.

═══ ЧТО НЕ ДЕЛАТЬ ═══

- НЕ меняй логику verify_vip_token / course_token_get_tier
- НЕ меняй формат токена / Ed25519-ключи
- НЕ занимайся token-first flow (выносом проверки выше меню) —
  это отдельный тикет, другой бриф
- НЕ редактируй ничего вне функции acquire_course_token_for_install
  (кроме INSTALLER_VERSION)

═══ КАК ПРОВЕРИТЬ ЧТО РАБОТАЕТ ═══

После применения прогони установщик с «грязным» токеном:

  bash scripts/demo-install.sh

В R0 введи (с пробелами в начале/конце):
  «  VIP-AAAAAAAAAAAAAAAA-12345  »

Должен напечататься «ℹ️ Очистил токен от лишних символов» —
после чего уже идёт обычная проверка подписи (которая упадёт
на тестовом токене, но это ОК — главное санитизация сработала).

═══ REFERENCE ═══

Готовый код в соседнем репо (1-в-1):
https://github.com/tonytrue92-beep/openclaw-agents-pack/blob/v2026.05.15/scripts/lib/course-token.sh

Поиск в файле: «Wave 17: санитизация ввода» — это начало блока.

END PROMPT ───────────────────────────────────────────────────────
```

## Что делает технарь после AI-coder

1. AI-coder создаёт PR
2. Технарь делает code review (1 минута — изменение тривиальное)
3. Технарь мержит PR в main
4. Тегает релиз (если в factory есть тег-based CI — auto-deploy)
5. Пишет Антону «готово, версия X в main» — Антон уведомляет Надежду

## Если AI-coder не справился

Fallback — ручной патч. См. `handoff/wave-17-factory-port-brief.md`
в этом же репо — там пошаговая инструкция «найди → вставь → закоммить»
с конкретными строками кода. Это займёт у технаря 15 минут вручную.

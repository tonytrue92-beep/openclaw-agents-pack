# ТЗ: SUB-tier (subscription) в `@AITeamVIPBot` v4

**Дата:** 2026-05-11
**Кому:** технарь Антона (`@AITeamVIPBot` + `openclaw-factory`)
**Автор запроса:** Антон Поляков
**Зависимости:** wave 12 уже сделано (STD/VIP в боте + factory)

## TL;DR

Добавляем **третий тариф** — `SUB` (subscription / подписка). Это
**самый базовый** тариф, ниже Standard:
- **SUB**: OpenClaw + main-агент (через первый установщик)
- **STD** (как раньше): + 3 агента (Технарь / Маркетолог / Продюсер)
- **VIP** (как раньше): + 6 агентов

У SUB-подписчиков `expires_at` — подписка ежемесячная, истекает.

CSV с 41 текущим подписчиком прилагаю — нужно залить первой партией.

## Что уже сделано (на стороне agents-pack)

**v2026.05.07.1** (wave 16 готовится) — установщик уже:
- Распознаёт `SUB-...` префикс через `vip_token_version()` → `v3-sub`
- `course_token_get_tier()` возвращает `SUB`
- `verify_vip_token` принимает SUB через `_verify_v3` с payload
  `SUB|<hash>|<tg>`
- На SUB-tier установщик **graceful-exit** с info-сообщением:
  > У тебя SUB-тариф (подписка). Базовая установка уже выполнена
  > первым установщиком — main-агент в Telegram отвечает.
  > Дополнительные агенты доступны на Standard/VIP. Апгрейд → саппорт.

Это значит **наш agents-pack уже готов** принять SUB-токены — нужен
твой апдейт бота + factory чтобы клиенты могли реально получить токен.

## Что нужно от тебя — 2 части

### Часть 1 — `@AITeamVIPBot` v4

#### 1.1 Новая таблица `subscribers`

Аналогично `vip_clients` и `standard_clients` из wave 12. Схема:

```sql
CREATE TABLE IF NOT EXISTS subscribers (
  email TEXT PRIMARY KEY,
  subscription_started_at TIMESTAMP WITH TIME ZONE,
  subscription_expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
  tg_user_id BIGINT,  -- заполняется при первом /start
  activated_at TIMESTAMP WITH TIME ZONE,
  imported_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
CREATE INDEX idx_subscribers_expires ON subscribers(subscription_expires_at);
```

#### 1.2 Команда `/admin_upload_subscribers`

Аналог `/admin_upload_std`. CSV-формат:

```csv
email,subscription_started_at,subscription_expires_at
123boss32194@mail.ru,2026-05-01 05:45:17.195886+00,2026-06-03 11:17:18.077+00
...
```

CSV `active_subscribers.csv` прилагается (41 запись на 2026-05-11).

#### 1.3 Поиск при `/start` (новая логика)

Порядок проверки:
```
1. vip_clients      → если найден → выдать VIP-...
2. standard_clients → если найден → выдать STD-...
3. subscribers      → если найден И NOW() < subscription_expires_at →
                       выдать SUB-...
4. Не найден ни в одной → отказ
```

**Важно**: если в `subscribers` нашли но `subscription_expires_at < NOW()`
— **специальное сообщение** клиенту:

```
✗ Подписка истекла {{дата}}.

Чтобы продолжить — продли подписку на странице курса:
   {{ссылка на оплату}}

После оплаты вернись сюда — токен выдам автоматически.
```

#### 1.4 v4 payload format

```
SUB|<email_hash16>|<tg_user_id>
```

Тот же что v3 (TIER + hash + tg), только TIER=`SUB`. Подпись Ed25519
тем же приватным ключом — публичный ключ в agents-pack уже
содержит проверку через `_verify_v3` для TIER=SUB.

**Никаких отдельных ключей не нужно.** Один приватный ключ для всех
3 тиров.

#### 1.5 Telemetry / алерты

Опционально на будущее:
- Лог активации с tier-разделением (для статистики)
- Алерт админу когда у клиента expires < 7 дней (для напоминаний)
- Алерт о массовых отказах «подписка истекла» (если CSV не обновили)

### Часть 2 — `openclaw-factory/scripts/demo-install.sh`

#### 2.1 SUB-token валидация

Сейчас (после wave 12) первый установщик валидирует STD/VIP. Нужно
добавить SUB:
- Распознавать `SUB-` префикс (так же как `STD-` / `VIP-`)
- Валидировать подпись через тот же `verify_vip_token` (наш `vip.sh`
  уже умеет SUB после wave 16)
- Кэш в `~/.openclaw/course-token` — единый для всех тиров

#### 2.2 SUB-режим установки

Для SUB-tier — **установить только базу**:
- OpenClaw движок (как раньше)
- main-агент (как раньше)
- **Не предлагать** «давай ещё запустим agents-pack» в финале
  (или предложить с уточнением «у тебя SUB-тариф, для доп. агентов
  нужен апгрейд»)

Информативное сообщение клиенту:
```
✓ Установка завершена!

Твой тариф: SUB (подписка)
  — OpenClaw движок ✓
  — main-агент в Telegram ✓

Дополнительные агенты (Технарь / Маркетолог / Продюсер и т.д.) —
на тарифах Standard / VIP. Апгрейд → саппорт-чат курса.

Подписка истекает: {{дата из токена или из бота}}
```

## Не делай (out of scope)

- **Auto-renewal интеграция с Prodamus** — wave 17+ если потребуется.
  Сейчас Антон вручную загружает CSV.
- **Email-уведомления когда expires < 7 дней** — отдельная задача,
  возможно через Telegram-уведомление вместо email.
- **Принудительное отключение агентов после истечения** — мы НЕ
  отключаем установленные локально агенты (это машина клиента,
  у него полный контроль). Только новая установка / refresh
  блокируется при отсутствии валидного токена.
- **Refund / cancellation logic** — на стороне платёжной системы.

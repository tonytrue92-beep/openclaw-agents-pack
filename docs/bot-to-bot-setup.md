# 🤖↔🤖 Bot-to-Bot Communication — настройка для VIP-команды

> **Что это:** новая фича Telegram (май 2026) — боты могут отвечать
> другим ботам напрямую, без посредника-человека. Для нашей VIP-команды
> из 6 агентов это даёт **прямые цепочки делегирования** (Координатор
> пишет Маркетологу → тот Копирайтеру → тот возвращает результат
> Координатору) **без** общей группы.
>
> 📖 Источник: [Telegram blog — AI Bot Revolution](https://telegram.org/blog/ai-bot-revolution-11-new-features/ru)
> 📖 Документация: [core.telegram.org/bots/features#bot-to-bot-communication](https://core.telegram.org/bots/features#bot-to-bot-communication)

---

## 🎯 Зачем это VIP-клиентам

**До wave 8 (group-mode):**
- Все 6 ботов — в общей TG-группе с тобой
- Тегаешь `@coordinator_bot` → он отвечает + тегает `@marketer_bot`
- Все сообщения видны тебе (как клиенту)

**С Bot-to-Bot Communication (новое):**
- Координатор может **напрямую** написать Маркетологу (без группы)
- Цепочка делегирования происходит «за кулисами»
- Тебе приходит **финальный результат** от того бота кому ты задал вопрос

**Когда что использовать:**

| Сценарий | Group-mode (wave 8) | Bot-to-Bot (новое) |
|---|---|---|
| Хочу видеть весь процесс | ✅ Да | ❌ Нет |
| Хочу только финальный результат | ❌ много сообщений | ✅ Да |
| Брейншторм (3-4 агента вместе) | ✅ удобно | ⚠️ только цепочка |
| Быстрая делегация | ⚠️ нужно тегать | ✅ автоматически |

**Можно использовать оба одновременно** — они не конфликтуют.

## ⚠️ Важное предупреждение про loops

Telegram официально требует от разработчика:

> Your bot must remain stable even if another bot intentionally
> responds instantly and continuously.

**Что это значит:** если 2 бота начали отвечать друг другу
бесконечно — Telegram может ограничить твоего бота. Защита от
зацикливания — твоя ответственность.

**Что мы уже сделали (на стороне agents-pack):**
- ✅ Все шаблоны агентов содержат правило `requireMention=true` —
  бот отвечает только когда явно тегнут
- ✅ Координатор имеет правило в `LEARNING.md`: «не отвечай на
  сообщения других ботов автоматически — только на явные запросы
  от пользователя»
- ✅ OpenClaw Gateway имеет встроенный rate-limiter (~1 ответ
  в N секунд)

**Что в твоей зоне:**
- Если боты начали зацикливаться — отключи Bot-to-Bot Mode у
  одного из них (см. секцию «Откат» ниже)
- Не давай агентам инструкции типа «постоянно проверяй у X не
  готово ли» — это рецепт зацикливания

## 🛠 Настройка — пошагово

### Шаг 1 — Включи Bot-to-Bot Mode для каждого бота

Telegram пока **не задокументировал точную команду** в @BotFather
(фича только что вышла). Действуй так:

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь `/mybots`
3. Выбери своего первого бота (например `@anton_coordinator_bot`)
4. Нажми **«Bot Settings»** (Настройки бота)
5. Найди опцию вида:
   - **«Bot-to-Bot Communication Mode»**
   - или **«Allow Bot Messages»**
   - или **«Talk to Other Bots»**
6. Включи (Enable / Turn On)

`[SCREENSHOT: BotFather → Bot Settings → Bot-to-Bot Mode]`

> 💡 **Если не нашёл опцию:** возможно Telegram ещё раскатывает
> фичу не на всех клиентов. Проверь:
> - Обнови приложение Telegram до последней версии
> - Попробуй на Telegram Desktop (туда обычно фичи приходят первыми)
> - Если совсем не находишь — напиши [@BotSupport](https://t.me/botsupport)

7. **Повтори для каждого** из 6 ботов VIP-команды:
   - `@anton_tech_bot`
   - `@anton_marketer_bot`
   - `@anton_producer_bot`
   - `@anton_designer_bot`
   - `@anton_coordinator_bot`
   - `@anton_copywriter_bot`

### Шаг 2 — Проверь что работает

Открой приватный чат с **Координатором** и напиши:

```
Скажи Маркетологу подобрать 3 канала для поста про мою нишу
```

**Что должно случиться:**
- Координатор отвечает тебе: «Передаю задачу @anton_marketer_bot...»
- Через 5-15 секунд Маркетолог пишет тебе **в ваш приватный чат с
  Координатором**: «Вот 3 канала: ...»

> 💡 Если Маркетолог не появился в этом чате — значит Bot-to-Bot
> ещё не работает. Проверь что у обоих ботов включён режим (Шаг 1).

### Шаг 3 — (Опционально) Можешь оставить group-mode тоже

Если ты раньше настраивал общую TG-группу через
`--enable-group-mode` (wave 8) — она **продолжит работать**.
Bot-to-Bot не отменяет group-mode.

Используй то что удобнее для конкретной задачи:
- **Группа** для брейнштормов где хочешь видеть мнения нескольких
  агентов
- **Bot-to-Bot** для быстрой делегации без шума

## 🛑 Откат (если что-то пошло не так)

### Сценарий: боты зациклились

Если видишь что 2 бота отвечают друг другу бесконечно (>5
сообщений за минуту без вмешательства человека):

1. **Открой @BotFather → /mybots → выбери одного из «зациклившихся»**
2. Bot Settings → выключи **Bot-to-Bot Mode** (Disable)
3. Зайди в чат → нажми **«Block Bot»** временно (на 1 минуту)
4. Через минуту — Unblock
5. Включай Bot-to-Bot обратно когда поймёшь причину loop'а
   (обычно это инструкция в SOUL/LEARNING.md одного из агентов)

### Сценарий: бот отвечает слишком много

Возможно clear `MEMORY.md` агента содержит инструкции «после каждого
ответа спрашивай у X». Найди и убери. Или используй
`--refresh-templates` чтобы вернуть характер агента к стандартному:

```bash
bash <(curl -fsSL https://github.com/tonytrue92-beep/openclaw-agents-pack/releases/latest/download/install-agents-bundled.sh) --refresh-templates
```

Это не тронет MEMORY/USER, но обновит SOUL/LEARNING до текущей
стабильной версии.

## ❓ FAQ

**Q: Это работает только для VIP или для Standard тоже?**
A: **Только для VIP.** Standard — это 3 агента (Технарь, Маркетолог,
Продюсер) которые работают **независимо**. Делегация цепочек —
концепция VIP (где есть Координатор как «менеджер»).

**Q: Сколько агентов могут общаться одновременно?**
A: Технически — все 6 одновременно. Практически — зависит от твоего
тарифа OpenAI / Anthropic (rate-limit на токены). Обычно безопасно
до 3-4 параллельных агентов.

**Q: Это безопасно? Не утечёт ли моя информация?**
A: Безопасно если все боты — твои (созданные тобой через @BotFather).
Bot-to-Bot работает только между ботами **одинакового пользователя
по аккаунту owner'а**. Чужой бот не сможет инициировать чат с
твоим (он просто не получит ответ).

**Q: Можно ли это сделать на Standard для будущего апгрейда?**
A: Можно — Bot-to-Bot Mode не повредит Standard-боту, просто
не используется. Если потом купишь VIP — настройка уже на месте.

**Q: А есть ли это в Bot API уже?**
A: Да, см. [core.telegram.org/bots/features#bot-to-bot-communication](https://core.telegram.org/bots/features#bot-to-bot-communication).
OpenClaw Gateway адаптируется к новому Bot API автоматически —
никаких apt-get / brew не надо.

**Q: Что если разные боты у меня и у моего сотрудника?**
A: Bot-to-Bot работает **внутри** одного аккаунта. Если у тебя
свои 6 ботов, у сотрудника — свои 6, они **не пересекаются**.

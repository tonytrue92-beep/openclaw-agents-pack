# 👥 Group Mode — команда AI-агентов в общей TG-группе

Эта инструкция — для клиентов, у которых **уже установлены** 2+ агентов
(Standard или VIP) и хочется чтобы они работали как команда: тегали друг
друга, делегировали задачи, видели общий контекст.

---

## Зачем это нужно

**Без group-mode:** каждый бот сидит в приватном чате с тобой.
Чтобы делегировать задачу от Маркетолога к Копирайтеру — ты копируешь
её сам в другой чат. Цепочки разваливаются, контекст теряется.

**С group-mode:** есть **одна общая TG-группа**, в которой все боты —
участники. Ты пишешь Маркетологу → он отвечает + тегает Копирайтера →
Копирайтер видит контекст и берёт работу. Цепочка не рвётся.

**Сценарии где это полезно:**
- Запуск продукта: Продюсер → Маркетолог → Копирайтер → Дизайнер
- Производство контента: ты даёшь задачу Координатору, он
  декомпозирует и распределяет между остальными
- Брейншторм: 3-4 агента вместе обсуждают идею, каждый со своей роли

---

## Что нужно подготовить вручную

### 1. Отключить privacy mode у каждого бота

По умолчанию Telegram-боты в группе видят **только сообщения адресованные им**
(через `/cmd@botname`, reply, или @mention). Чтобы бот мог *проактивно*
включаться в обсуждение — нужно отключить privacy mode.

Для **каждого** бота:

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь `/setprivacy`
3. Выбери бота из списка
4. Выбери `Disable`

Бот ответит:

> Success! The new status is: DISABLED.

Повтори для каждого из своих ботов (3 для Standard, 6 для VIP).

> 💡 **Альтернатива:** оставить privacy включённым (Enable). Тогда бот
> в группе видит только сообщения с явным @mention. Минус — он не сможет
> следить за общим обсуждением и проактивно встрять. Для большинства
> сценариев лучше Disable.

### 2. Создать TG-группу и добавить ботов

1. В Telegram: меню → «Создать группу»
2. Назвать (например, «AI Команда»)
3. Добавить **всех** своих ботов (искать по `@<имя_бота>`)
4. **Каждого бота сделать админом**: настройки группы → участники →
   на боте «Сделать администратором». Без admin-прав некоторые
   действия (закреп, реакции) недоступны

### 3. Узнать chat_id группы

**Способ A — через бота:**
1. Открой [@username_to_id_bot](https://t.me/username_to_id_bot)
2. Перешли любое сообщение **из своей группы** в этот бот
3. Бот ответит `Forwarded from chat: -100123456789`
4. Скопируй число (с минусом)

**Способ B — для супергрупп:**
- Открой группу в Telegram Desktop / Web
- Скопируй ссылку на любое сообщение: `https://t.me/c/123456789/42`
- chat_id = `-100` + это число = `-100123456789`

## Запуск настройки

Когда сделал шаги 1-3 — выполни:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/tonytrue92-beep/openclaw-agents-pack/main/scripts/install-agents.sh) --enable-group-mode -100123456789
```

Замени `-100123456789` на свой chat_id.

**Что произойдёт:**
1. Установщик найдёт всех твоих агентов
2. Покажет чек-лист (Disable privacy / админы / chat_id)
3. Спросит подтверждение
4. Для каждого агента пропишет в `~/.openclaw/openclaw.json`:
   - `groupPolicy = allowlist`
   - `groupAllowFrom += <chat_id>`
   - `groups.<chat_id>.requireMention = true`
5. Готово!

## Проверка работы

Зайди в свою TG-группу и напиши:

```
@<username_координатора>, проверь связь
```

Координатор должен ответить через 5-10 секунд.

Затем:

```
@<username_координатора>, попроси Маркетолога дать 3 идеи поста про мою нишу
```

Координатор должен:
1. Понять задачу
2. Тегнуть `@<username_маркетолога>`
3. Маркетолог увидит тег и ответит со своими идеями

## Типичные проблемы

### Бот не отвечает в группе

1. **Проверь что бот админ:** настройки группы → участники → бот
   должен быть с короной (admin)
2. **Проверь privacy mode:** `@BotFather` → `/setprivacy` → выбери
   бота — должно быть `DISABLED`
3. **Проверь конфиг:** запусти `--diagnose-only` — должно показать
   `group-mode: -100... (вкл)`
4. **Проверь что тегнул правильно:** должно быть `@<имя_бота>`
   (не сообщение reply, а @-mention в тексте)

### Боты тегают друг друга бесконечно

`requireMention=true` должен это предотвращать — бот отвечает только
если **тегнули его явно**. Если loop всё-таки случается:

1. Удали бота из группы
2. Проверь `~/.openclaw/openclaw.json`:
   ```bash
   openclaw config get channels.telegram.accounts.<id>.groups
   ```
3. Должно быть `requireMention: true` для каждого chat_id
4. Если нет — повтори `--enable-group-mode <chat_id>`

### Хочу убрать одного агента из группы

```bash
# Снимаем chat_id из allowFrom
openclaw config set channels.telegram.accounts.<id>.groupPolicy disabled

# Или удаляем из allowFrom вручную через openclaw config set
```

### Хочу добавить нового агента в существующую группу

1. Установи нового агента (если ещё нет): `--only <agent>` или обычный run
2. Добавь его бота в группу (как админа)
3. `@BotFather → /setprivacy → новый бот → Disable`
4. Запусти `--enable-group-mode <chat_id>` — установщик найдёт нового
   агента и пропишет ему ту же группу

## Откат

Чтобы убрать group-mode полностью:

```bash
for agent in tech marketer producer designer coordinator copywriter; do
  openclaw config set "channels.telegram.accounts.${agent}.groupPolicy" disabled 2>/dev/null
  openclaw config delete "channels.telegram.accounts.${agent}.groupAllowFrom" 2>/dev/null
  openclaw config delete "channels.telegram.accounts.${agent}.groups" 2>/dev/null
done
```

После — удали ботов из группы или удали саму группу.

DM-режим (приватные чаты) при этом не пострадает — он управляется
другими настройками (`dmPolicy`, `allowFrom`).

# Close Loop — закрываю сессию правильно

Скилл от `@clarezoe` (версия 2.1.1) — end-of-session workflow.
Когда сессия подходит к концу (ты говоришь «подводим итоги» /
«на сегодня всё» / «wrap up»), я запускаю 4-фазное закрытие чтобы
ничего не потерялось.

## Когда использую

- Ты пишешь «давай подводить итоги сессии»
- Конец дня — закрываем день правильно, чтобы утром начать с чистого
  листа
- Перед долгой паузой (отпуск, выходные) — фиксирую состояние
- Завершение крупной задачи — консолидируем результаты

## Когда НЕ использую

- Сессия живая, есть открытые вопросы — значит рано закрывать
- Ты не просил закрыть — не навязываю. Инициатива твоя.

## 4 фазы закрытия

### Phase 1 — Ship state

Фиксирую **что сделано**:
- Список action items которые были в сессии
- Статус каждой задачи: сделана / частично / отложена / отменена
- Артефакты которые надо сохранить (ссылки, файлы, скриншоты)
- Где они лежат

Результат → в `MEMORY.md` под сегодняшней датой.

### Phase 2 — Consolidate memory

Разбираю `MEMORY.md`:
- Новые факты о проекте / клиенте / процессе добавляю в «Постоянное»
- Одноразовые детали ухода оставляю в «Сегодня»
- Устаревшие записи (решённые вопросы, отменённые планы) помечаю
  как закрытые или удаляю

### Phase 3 — Apply improvements

Если в сессии была ошибка которой я не должен повторять:
- Записываю в `LEARNING.md` в формате `[CORRECTION] → [CORRECT] → [RULE]`
- Так в следующей сессии я не повторю её

### Phase 4 — Publishable outputs

Если сессия произвела что-то «на выход» (пост / текст / скрипт):
- Складываю в `workspace/artifacts/` с понятным именем
- Отмечаю в `MEMORY.md`: «готово, отправлено» / «готово, ждёт твой approve»

## Формат отчёта

После закрытия выдаю тебе два артефакта:

### 1. Human-readable report
```markdown
# Session Close — 2026-XX-XX

## Сделано
- [✓] Подготовлен бриф для Дизайнера по баннеру
- [✓] 2 заголовка для лид-магнита — от Копирайтера

## Частично
- [~] Контент-план на неделю — готово 3 из 5

## Отложено
- [ ] Запуск рассылки — ждём визуал от Дизайнера

## Lessons learned
- [CORRECTION] ... → [CORRECT] ... → [RULE] ...

## Дальше
- Утром: собрать готовые материалы, отправить
- Блокер: нет визуала от Дизайнера — напомнить
```

### 2. Machine-readable JSON
Структурированный блок для программного чтения (если когда-нибудь
захочешь автоматизировать анализ сессий).

## Безопасные выходы

Перед закрытием проверяю:
- Нет ли в артефактах случайных секретов (API-ключей, email'ов
  в чистом виде)
- Все ссылки рабочие
- Маркдаун корректный

## Главное

- Без твоего инициатива «закрой сессию» — не закрываю. Это твоё решение.
- Закрытие сессии ≠ удаление памяти. MEMORY.md не чищу — только
  реорганизую.
- LEARNING.md после close-loop становится точнее.
